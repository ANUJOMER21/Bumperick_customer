Complete Setup Guide

Detailed step-by-step instructions to get BumperPick running on your development environment

1

Prerequisites & Environment Setup

Ensure you have all required tools and SDKs installed before proceeding

  • Android Studio Arctic Fox (2020.3.1) or later
  • JDK 8 or higher (JDK 11 recommended)
  • Android SDK API Level 28-34
  • Kotlin 2.0.0
  • Gradle 8.7.3
  • Git for version control
# Check Java version
java -version

# Check Android SDK
sdkmanager --list
2

Clone Repository

Get the source code from the GitHub repository

# Clone the repository
git clone https://github.com/ANUJOMER21/Bumperick_customer.git
cd Bumperick_customer

# Checkout the main branch
git checkout main
3

Configure Google Maps API

Set up Google Maps integration for location services

  • Visit Google Cloud Console
  • Create a new project or select existing one
  • Enable Maps SDK for Android API
  • Enable Places API
  • Create API key with proper restrictions
  • Add API key to AndroidManifest.xml
# In app/src/main/AndroidManifest.xml
<meta-data
    android:name="com.google.android.geo.API_KEY"
    android:value="YOUR_API_KEY_HERE" />
4

Firebase Configuration

Set up Firebase for cloud messaging and analytics

  • Create Firebase project at console.firebase.google.com
  • Add Android app with package name: com.bumperpick.bumperickUser
  • Download google-services.json
  • Place file in app/ directory
  • Enable Firebase Cloud Messaging
  • Enable Firebase Analytics
# File structure should look like:
app/

‚îú‚îÄ‚îÄ build.gradle.kts
‚îî‚îÄ‚îÄ src/main/

API Configuration

Configure the backend API endpoint and authentication

# Update app/src/main/java/com/bumperpick/bumperickUser/DI/AppModule.kt
single {
    Retrofit.Builder()
        .baseUrl("http://65.0.78.200/") // Your API URL
        .addConverterFactory(GsonConverterFactory.create())

}
6

Build Configuration

Verify build configuration and sync dependencies

# Sync project with Gradle files
./gradlew clean
./gradlew build

# Or use Android Studio:
# File ‚Üí Sync Project with Gradle Files
7

Run Application

Deploy the app to your device or emulator

  • Connect Android device with USB debugging enabled
  • Or start Android emulator
  • Run the application from Android Studio
  • Or use command line build
# Command line build and install
./gradlew assembleDebug
./gradlew installDebug

# Or build release version
./gradlew assembleRelease
8

Testing & Verification

Verify all features are working correctly

  • Test phone number authentication
  • Verify Google Sign-in functionality
  • Check location services and maps
  • Test offers and events loading
  • Verify push notifications
  • Test contest registration
# Run unit tests
./gradlew test

# Run instrumented tests
./gradlew connectedAndroidTest

API Documentation

Comprehensive API reference for BumperPick backend integration

üîê Authentication APIs

POST /api/customer/send-otp

Send OTP to customer's phone number for authentication

Request: { "mobile": "+1234567890" }
Response: { "success": true, "message": "OTP sent" }
POST /api/customer/verify-otp

Verify OTP and authenticate user

Request: { "mobile": "+1234567890", "otp": "123456" }
Response: { "success": true, "token": "jwt_token", "user": {...} }
POST /api/customer/auth-google

Authenticate user with Google Sign-in

Request: { "idToken": "google_id_token" }
Response: { "success": true, "token": "jwt_token", "user": {...} }

üéØ Offers APIs

POST /api/customer/offers

Get offers list with filtering and pagination

Request: { "page": 1, "limit": 20, "category_id": 1 }
Response: { "offers": [...], "pagination": {...} }
GET /api/customer/offers-details/{id}

Get detailed information about specific offer

GET /api/customer/offers-history

Get user's offer usage history
